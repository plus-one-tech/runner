# Codex 向け実装指示文

次の仕様書を唯一の正とみなして実装してください。

* `docs/runner-spec.md`

この仕様に基づいて、Go で `runner` を実装してください。

## 実装方針

* まず **受け入れテストを先に作成**してください
* その後、テストを通す最小限の実装を行ってください
* 仕様に書かれていない便利機能は追加しないでください
* shell は使用しないでください
* 実行は必ず **直接実行**としてください
* diff は小さく保ってください
* リファクタリングは、テスト通過後に必要最小限だけ行ってください

## 実装対象

以下を実装してください。

1. `runner [options] [target]` の CLI
2. target 解決
3. `.run` ファイルの読み込みとヘッダ解析
4. `runner.env` の読み込み
5. runtime 解決
6. runtime command の引数分割
7. `.run` 本文の一時ファイル展開
8. 子プロセス実行
9. `--dry-run`
10. `--help`
11. `--version`
12. `--list`
13. 終了コード伝播
14. 仕様に定義されたエラー処理

## テスト方針

* **単体テスト**だけでなく、CLI の **結合テスト**を作成してください
* 実際に `runner` を子プロセスとして起動し、

  * 標準出力
  * 標準エラー
  * 終了コード
  * 一時ファイルの有無
    を確認してください
* OS 差が出る部分は、仕様で許可された範囲で吸収してください
* `UTF-8`, `UTF-8 BOM`, `LF`, `CRLF` のケースをテストしてください

## command 分割ルール

`runner.env` の `<command>` は以下の仕様で分割してください。

* 空白で分割する
* ダブルクォート `"..."` で囲まれた部分は1つの引数として扱う
* エスケープは `\"` と `\\` のみ有効
* 不正なクォートはエラー
* shell 展開は行わない
* shell を経由せず、argv として直接実行する

## 一時ファイル

* `.run` 本文は OS の一時ディレクトリに展開してください
* 実行終了後に削除してください
* dry-run では実行しないでください
* 仕様に反しない範囲でベストエフォートに後始末してください

## 実装の進め方

以下の順で進めてください。

1. 受け入れテスト一覧をテストコードへ落とし込む
2. failing test を作る
3. 最小実装で通す
4. 必要最小限の整理を行う

## 禁止事項

* 仕様にない機能追加
* shell 実行
* 独自 DSL の追加
* 過剰な抽象化
* 大規模リファクタリングの先行実施

## 期待する成果物

* Go 実装コード
* テストコード
* 必要最小限の README 更新
* 実装上の判断が必要だった箇所のメモ

## 出力形式

作業は次の順で提示してください。

1. 実装計画
2. 追加するテスト一覧
3. 実装 diff
4. テスト結果
5. 残課題
