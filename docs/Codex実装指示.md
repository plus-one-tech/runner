# Codex 向け実装指示文

次の仕様書を唯一の正とみなして実装してください。

* `docs/runner-spec.md`

この仕様に基づいて、Go で `runner` を実装してください。

## 実装方針

* まず **受け入れテストを先に作成**してください
* その後、テストを通す最小限の実装を行ってください
* 仕様に書かれていない便利機能は追加しないでください
* shell は使用しないでください
* 実行は必ず **直接実行**としてください
* diff は小さく保ってください
* リファクタリングは、テスト通過後に必要最小限だけ行ってください

## 実装対象

以下を実装してください。

1. `runner [options] [target]` の CLI
2. target 解決
3. `.run` ファイルの読み込みとヘッダ解析
4. `runner.env` の読み込み
5. runtime 解決
6. runtime command の引数分割
7. `#script` / OS ブロック解析
8. `var.*` の変数展開
9. `.run` 本文の一時ファイル展開
10. 子プロセス実行
11. `--dry-run`
12. `--help`
13. `--version`
14. `--list`
15. 終了コード伝播
16. 仕様に定義されたエラー処理

## テストデータの利用

実装は、提供された testdata ディレクトリを使用して検証すること。

testdata に含まれるファイルを用いて、受入テスト仕様の各テストケースを再現できること。

### テストデータ対応

* script-duplicate-os.run → AT-05A
* script-unknown-os.run → AT-05B
* script-no-os-block.run → AT-05C
* script-missing-runtime.run → AT-058
* script-invalid-outer-body.run → AT-059

## golden ファイルとの一致

#script モードにおいて生成されるスクリプトは、
testdata/golden 配下のファイルと一致すること。

比較は以下の条件で行う:

* 改行コードの差異は許容する（LF / CRLF）
* それ以外は完全一致とする

## エラーケース

以下の testdata ファイルはエラーになること:

* script-duplicate-os.run
* script-unknown-os.run
* script-no-os-block.run
* script-missing-runtime.run
* script-invalid-outer-body.run

エラー時は、受入テスト仕様に記載されたメッセージを出力すること。

## OS依存テスト

以下のファイルは OS に依存する:

* script-side-effects.run
* script-windows-only.run
* script-linux-only.run

実行環境の OS に応じて、該当するブロックのみが実行されること。

## runner.env の適用

テストでは `--env testdata/env/runner.env` を指定して実行すること。
runner.env の読み込みは仕様に従い、`--env` によって明示指定された場合のみ使用する。

特に以下を確認すること:

* runtime の解決
* ext の解決
* var の展開

## dry-run の検証

dry-run 実行時は以下を検証対象とする:

* 実行コマンドが正しいこと
* OS 判定が正しいこと
* 展開スクリプトが正しいこと

## テスト方針

* **単体テスト**だけでなく、CLI の **結合テスト**を作成してください
* 実際に `runner` を子プロセスとして起動し、

  * 標準出力
  * 標準エラー
  * 終了コード
  * 一時ファイルの有無
    を確認してください
* OS 差が出る部分は、仕様で許可された範囲で吸収してください
* `UTF-8`, `UTF-8 BOM`, `LF`, `CRLF` のケースをテストしてください

## command 分割ルール

`runner.env` の `<command>` は以下の仕様で分割してください。

* 空白で分割する
* ダブルクォート `"..."` で囲まれた部分は1つの引数として扱う
* エスケープは `\"` と `\\` のみ有効
* 不正なクォートはエラー
* shell 展開は行わない
* shell を経由せず、argv として直接実行する

## `#script` / 変数展開

* `#script` は `.run` の特別ヘッダとして実装してください
* `@windows` / `@linux` / `@macos` の OS ブロックを解析してください
* OS ブロック内の最初の非空行は `#<runtime>` 必須です
* `#script` の dry-run は全 OS ブロックを順に表示してください
* 変数展開は `runner.env` の `var.*` に対する `${var.*}` のみ対応してください
* OS 環境変数は展開しないでください

## 一時ファイル

* `.run` 本文は OS の一時ディレクトリに展開してください
* `#script` では選択された OS ブロックの本文だけを展開してください
* 実行終了後に削除してください
* dry-run では実行しないでください
* 仕様に反しない範囲でベストエフォートに後始末してください

## 実装の進め方

以下の順で進めてください。

1. 受け入れテスト一覧をテストコードへ落とし込む
2. failing test を作る
3. 最小実装で通す
4. 必要最小限の整理を行う

## 禁止事項

* 仕様にない機能追加
* shell 実行
* 独自 DSL の追加
* 過剰な抽象化
* 大規模リファクタリングの先行実施

## 期待する成果物

* テストコード
* 必要最小限の README 更新
* 実装上の判断が必要だった箇所のメモ
