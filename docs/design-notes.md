# runner 検討メモ

## 1. ファイル探索の扱い

### 現状
* runner は指定されたファイル名をそのまま扱う
* 拡張子の補完は行わない

### 論点
* `runner build` のように拡張子なしで呼べた方が便利ではないか
* 特に Linux では拡張子なし文化との相性がよい

### 考えた案
#### 案1: 現状維持
* `runner build.run` のように明示指定を必須とする

#### 案2: `.run` を補完する
* `runner build` の場合に `build.run` を探索する

#### 案3: `.run` と拡張子なしの両方を探索する
* 探索順を決めて補完する

### 今回の判断
* 現バージョンでは現状維持
* まずは現行仕様を明確化することを優先する
* 探索順の変更は将来バージョンの検討事項とする

### 判断理由
* いま挙動を変えると仕様確定と実装変更が同時に走ってしまう
* まずは「現在どう動くか」を固定したい
* 探索順は UX に関わるため、次バージョンで独立して検討した方がよい

### 補足
* runner の主体は `.run` ファイルと考えている
* 拡張子なし対応を入れる場合でも、あくまで Linux 文化への補助対応として考える

## 2. スクリプト内容の書き出し仕様

### 現状
* 先頭行はヘッダとして扱う
* 2行目以降を実行対象スクリプトとして扱う想定

### 問題
* 2行目以降の内容を temp ファイルに正しく書き出す処理が未整理
* runtime によっては実行形式が異なるため、書き出し方式が影響を受ける

### 論点
* 改行の扱い（そのままか、調整するか）
* BOM やエンコーディングの扱い
* シェルにそのまま渡すか、一時ファイルにするか
* dry-run 時の扱い

### 現時点の判断
* 挙動を明確化するまでは仕様を固定しない
* 実装を先に安定させる

### 補足
* この部分は runner のコア処理にあたるため、後回しにしない

## 2. スクリプト内容の書き出し仕様

### 現状

* 先頭行はヘッダとして扱う
* 2行目以降を実行対象スクリプトとして扱う想定

### 問題

* 2行目以降の内容を temp ファイルに正しく書き出す処理が未整理
* runtime によっては実行形式が異なるため、書き出し方式が影響を受ける
* **1行目が言語にとって不正な構文になる可能性がある**

### 論点

* ヘッダ行を実行対象に含めるか除外するか
* 改行の扱い（そのままか、調整するか）
* BOM やエンコーディングの扱い
* シェルにそのまま渡すか、一時ファイルにするか
* dry-run 時の扱い

### 検討

* ヘッダをそのまま残す場合、言語ごとにコメント記法が異なるため整合が取れない
* ヘッダを除去する場合、全言語で一貫した挙動になる

### 現時点の判断

* ヘッダは実行対象に含めない前提で整理する
* 挙動を明確化するまでは仕様を固定しない
* 実装を先に安定させる

### 補足

* この部分は runner のコア処理にあたるため、後回しにしない
* runner は言語仕様に依存しない設計を優先する
